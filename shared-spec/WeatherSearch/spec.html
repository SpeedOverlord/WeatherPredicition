功能規格 · 平台中立 SSOT

天氣搜尋 WeatherSearch

輸入城市名稱，串接中央氣象署「一般天氣預報－今明 36 小時天氣預報」，於同一畫面以四種狀態呈現一或多個縣市的預報。同一份規格同時驅動 iOS（MVVM-C）與 Flutter（Bloc）。

v1.3 F-C0032-001 22 縣市 AC-1 ～ AC-9 iOS + Flutter 行為一致
01

四種顯示狀態

initial
初始
尚未搜尋，顯示輸入提示。不呼叫 API。
loading
讀取中
畫面內 inline loading，非 Dialog / 非彈窗。
loaded
氣象資料
一或多個縣市的可收合卡片；展開時 3 時段三欄並排、依天氣上底色。每次搜尋一律收合。
error
錯誤
查無城市 / 資料格式錯誤 / 連線失敗，各顯示對應訊息。

三層式泳道 · 搜尋流程

由左至右：畫面 → 邏輯 → 資料。數字為流程順序。

邏輯 · 狀態ViewModel / Cubit

1

輸入城市名稱 → 點「確認」

城市名可留空、可為部分名稱（如 台北、新竹）。

2

正規化輸入

去除前後空白，並將「台」統一為「臺」。

3

輸入解析 · CityNameResolver

以正規化字串比對 22 縣市清單，決定目標縣市。

空→ 全部 22 唯一前綴→ 補全 1 筆 多重前綴→ 多筆 無符合→ 查無城市 · 不呼叫 API
4

狀態 → 讀取中

顯示區塊切換為 inline loading。

5

呼叫 CWA API

不帶 locationName → 回傳全部縣市。

網路 / 401 / 5xx→ 連線失敗
6

解析所有 location → 縣市預報清單

合併 Wx / PoP / MinT / MaxT / CI 為 3 個時段。

無法解析 / 缺欄位→ 資料格式不正確
7

依目標縣市過濾

保留步驟 3 決定的縣市；若過濾後為空 → 查無城市。

8

顯示結果

氣象資料清單（縣市 × 3 時段）或對應錯誤訊息。

04

錯誤處理

三種錯誤路徑皆收斂到 inline 錯誤畫面。

cityNotFound
搜尋條件無效
查無此城市，請確認名稱後再試

非空輸入正規化後不符任何有效縣市前綴（如 火星市）。不呼叫 API。

↳ AC-5
invalidData
API 資料格式不正確
資料格式不正確，請稍後再試

JSON 無法解析成預期結構，或缺少必要氣象要素 / 時段數不對齊。程式不崩潰。

↳ AC-6
requestFailed
連線 / 伺服器錯誤
連線失敗，請檢查網路或稍後再試

網路中斷、逾時、HTTP 5xx。程式不崩潰。

↳ AC-7
unauthorized
授權失敗（HTTP 401，單獨）
授權失敗，請確認 API 授權碼

授權碼無效 / 未帶。與連線失敗分開，方便辨識金鑰問題。程式不崩潰。

↳ AC-10
05

API 端點

GET https://opendata.cwa.gov.tw/api/v1/rest/datastore/F-C0032-001
  • Authorization（隱含）— CWA 授權碼，存於 gitignored secrets 檔，不進版控。
  • locationName — 不帶；省略時回傳全部 22 縣市，縣市過濾由 App 於用戶端進行。
  • 回應要素 — 每縣市 3 個時段，每段合併以下 5 要素：
    Wx 天氣現象 PoP 降雨機率 MinT 最低溫 MaxT 最高溫 CI 舒適度
06

驗收條件

每條 AC ↔ 兩平台各一個鏡像測試。

AC-1剛進入畫面 → 呈現初始狀態，未呼叫 API。
AC-2完整縣市名（臺北市）→ 清單顯示該 1 個縣市，含 3 時段。
AC-3搜尋時狀態序列：讀取中 → 完成。
AC-4空白輸入 → 清單顯示全部 22 縣市。
AC-5無符合輸入（火星市）→ 查無城市，且未呼叫 API。
AC-6API 無法解析 / 缺欄位 → 資料格式不正確，不崩潰。
AC-7請求失敗（網路 / 逾時 / 5xx）→ 連線失敗，不崩潰。
AC-8唯一前綴（台北）→ 自動補全為臺北市，清單 1 筆。
AC-9多重前綴（新竹）→ 顯示新竹縣 + 新竹市。
AC-10HTTP 401（授權碼無效）→ 授權失敗（與連線失敗分開），不崩潰。
